# Предложение по адаптации системы TB-Expert для хакатона «Внедрение ИИ в сфере Охраны Труда»

---

## 1. Понимание задачи хакатона

Хакатон ставит задачу разработки **AI-аналитического модуля** поверх существующей HSE-системы (ИСОТ), включающей два модуля:

- **Модуль «Происшествия»** — учёт инцидентов с flash-отчётами (молниями), фотографиями, причинами и статусами.
- **Модуль «Карта Коргау»** — журнал поведенческих аудитов безопасности (хорошие и плохие практики на объектах).

Ожидаемый результат — **работающий AI-дашборд**, который:

1. Анализирует прошлые инциденты (классификация, паттерны, причины).
2. Предсказывает будущие риски (предиктивная аналитика на 3/6/12 месяцев).
3. Генерирует конкретные рекомендации по мерам контроля.
4. Показывает экономический эффект от внедрения мер безопасности — в людях и деньгах.
5. Обеспечивает систему алертов по данным Карт Коргау.

Технический стек предполагает: ETL-пайплайн, ML-ядро (NLP + предиктив + CV), дашборд с алертами и экспортом отчётов.

---

## 2. Существующая реализация: система TB-Expert

Система **TB-Expert** (Эксперт по ТБ) — это production-ready бэкенд для управления инцидентами в области охраны труда, разработанный для портфельных компаний Самрук-Қазына. Система уже реализует **значительную часть функциональности**, требуемой хакатоном.

### Близость задач

Задачи хакатона и текущей реализации совпадают по **ключевым направлениям**:

| Направление | Хакатон (ИСОТ) | TB-Expert (реализовано) |
|-------------|-----------------|------------------------|
| Учёт инцидентов | Модуль «Происшествия» (UUID, тип, дата, организация, локация, описание, причины, фото, статус) | Полная модель инцидента (58 полей), включая все требуемые + расширенные данные (стаж, оборудование, погодные условия, ответственные лица) |
| Импорт данных | HSE-система → ETL | Excel/CSV импорт журналов SAP, загрузка PDF/DOCX актов расследований с OCR и LLM-извлечением |
| NLP-анализ текстов | Классификация по типу, кластеризация причин | LLM-извлечение 15+ структурированных полей из текста актов, категоризация причин и нарушений |
| Аналитика и дашборд | Статистика, фильтрация, графики | Многомерная аналитика (регион, компания, тип, период), интерактивный дашборд с Chart.js |
| Рекомендации | AI-генерация мер контроля | LLM-генерация приоритизированных рекомендаций со ссылками на законодательство РК |
| Отчётность | PDF/DOCX/Excel экспорт | DOCX-экспорт аналитического отчёта с 7 разделами |

### Архитектурные преимущества

TB-Expert построен на **четырёхслойной Clean Architecture** с паттернами CQRS и Mediator:

```
Presentation (FastAPI) → Application (Commands/Queries) → Domain (Entities/Enums) → Infrastructure (DB/LLM/S3)
```

Это обеспечивает:
- **Модульность** — новые компоненты (Карта Коргау, предиктив, алерты) подключаются без переработки ядра.
- **Масштабируемость** — асинхронный I/O, DI-контейнер Punq, event-driven архитектура.
- **Расширяемость** — новые команды/запросы регистрируются в DI и автоматически маршрутизируются через медиатор.

## 3. Покрытие требований хакатона

### 3.1 Компонент A: AI-аналитика модуля «Происшествия»

#### F-01: Автоматическая классификация происшествий по типу на основе NLP-анализа
**Статус: Реализовано (адаптация минимальна)**

Система поддерживает 9 типов классификации инцидентов (`IncidentClassification`), включая НС, ДТП, аварии, пожары, инциденты на ОПО. При импорте из Excel классификация извлекается из исходных данных SAP. LLM-сервис извлечения (`LLMExtractionService`) уже анализирует текстовые описания актов расследований и присваивает структурированные категории.

**Для полного соответствия:** добавить NLP-классификатор, работающий по полю `description` инцидента для автоматического определения типа при создании записи.

---

#### F-02: Выявление паттернов по типу, организации, локации, времени
**Статус: Реализовано**

Эндпоинт `GET /v1/analytics/summary` возвращает агрегированную статистику в разрезах:
- По классификации (`by_classification`)
- По типу травмы (`by_injury_type`)
- По региону (`by_region`) — 21 регион Казахстана
- По компании (`by_company`) — 10 портфельных компаний
- По месяцам (`by_month`) — временная динамика

Дашборд визуализирует эти данные через Chart.js (круговые, столбчатые диаграммы, временные ряды).

---

#### F-03: Анализ причин инцидентов с кластеризацией по корневым причинам
**Статус: Реализовано**

- LLM-извлечение из актов расследований определяет `root_causes`, `immediate_causes`, `circumstances`.
- Тегирование по категориям: `cause_categories[]`, `violation_types[]`, `industry_tags[]` — массивы тегов, извлекаемых LLM из текста актов.
- Эндпоинт `GET /v1/acts/patterns` агрегирует теги причин и нарушений по частоте встречаемости.
- Аналитический отчёт содержит раздел **«Анализ причин»** с LLM-анализом корневых причин и паттернов повторяемости (`cause_analysis`, `top_cause_categories`, `recurrence_patterns`).

---

#### F-04: Генерация автоматических рекомендаций по мерам контроля
**Статус: Реализовано**

Аналитический отчёт (`POST /v1/analytics/report`) включает два раздела рекомендаций:

1. **Рекомендации** (`recommendations[]`) — приоритизированные меры с обоснованием, целевыми организациями, ссылками на НПА.
2. **Неотложные действия** (`immediate_actions[]`) — срочные меры, требующие немедленного исполнения.

Рекомендации генерируются LLM на основе:
- Результатов анализа причин (Phase 2)
- Оценки рисков (Phase 2)
- Релевантных норм законодательства РК (RAG-поиск по базе НПА)
- Данных об ответственных лицах и оборудовании

Сущность `Recommendation` имеет статусы отслеживания: «Ожидает выполнения» → «В работе» → «Выполнено» / «Отклонено».

---

#### F-05: Сводный дашборд со статистикой
**Статус: Реализовано**

Дашборд (`/demo`) реализован как SPA на vanilla JS + Chart.js:
- **KPI-карточки**: общее число инцидентов, пострадавших, погибших.
- **Фильтры**: компания, период, регион, классификация, тип травмы, текстовый поиск (27 параметров фильтрации).
- **Визуализации**: распределение по типам (pie), по регионам (heatmap), динамика по месяцам (line/bar).
- **Таблицы**: пагинированный список инцидентов и актов расследований.
- **Импорт данных**: drag-and-drop загрузка Excel/PDF/DOCX.
- **Генерация отчётов**: форма запуска с выгрузкой DOCX.

---

#### F-06: Предиктивная модель — прогноз на 3/6/12 месяцев
**Статус: Не реализовано (требуется разработка)**

Предиктивная аналитика (ARIMA / Prophet / LSTM) не входила в scope текущей задачи. Однако вся необходимая инфраструктура готова:

- **Данные**: исторические инциденты с полями `incident_date`, `company`, `region`, `classification`, `injury_type` уже структурированы в PostgreSQL с индексами по дате и компании.
- **Агрегация по месяцам** (`by_month`) уже реализована — это готовый временной ряд для обучения модели.
- **Архитектура**: новый Query-хендлер (`GetPredictionQuery`) подключается через медиатор без изменения существующего кода.
- **Оценка трудозатрат**: реализация базовой модели Prophet на существующих данных — **2-3 дня разработки**.

---

#### F-07: Расчёт экономического ущерба и экономии
**Статус: Частично реализовано (требуется расширение)**

Текущие данные содержат:
- `damage_amount_kzt` — фактический ущерб в тенге по каждому инциденту.
- `victim_count`, `fatality_count` — число пострадавших и погибших.
- `employer_fault_pct` — процент вины работодателя (из акта расследования).

**Для полного соответствия:** добавить калькулятор экономического эффекта на основе отраслевых коэффициентов (прямые затраты × 2 = косвенные, штрафы, простои). Формулы из раздела 8.2 ТЗ хакатона легко реализуются как детерминированный Python-модуль поверх существующей аналитики. **Оценка: 1-2 дня.**

---

#### F-08: Анализ фотографий с помощью CV-модели
**Статус: Частично реализовано (OCR есть, CV-детекция — нет)**

Система имеет `QwenOcrService` — сервис распознавания текста на изображениях через vision-модель Qwen. Инфраструктура (загрузка изображений в S3, обработка через LLM API) уже на месте.

**Для полного соответствия:** использовать ту же vision-модель для детектирования нарушений ТБ на фотографиях (отсутствие СИЗ, нарушения ограждений и т.д.). Приоритет в ТЗ хакатона: средний.

---

#### F-09: Экспорт аналитических отчётов в PDF/DOCX/Excel
**Статус: DOCX реализован, PDF и Excel — не реализованы**

`DocxReportBuilder` формирует профессиональный Word-документ с разделами: сводка, ключевые выводы, анализ причин, категории, паттерны, оценка рисков, рекомендации.

**Для полного соответствия:** PDF-экспорт добавляется через конвертацию DOCX → PDF (библиотека `docx2pdf` или LibreOffice headless). Excel-экспорт — через `openpyxl` (уже в зависимостях). Приоритет в ТЗ: низкий. **Оценка: 1 день.**

---

### 3.2 Компонент B: AI-аналитика «Карт Коргау»

#### F-10: Анализ паттернов нарушений
**Статус: Не реализовано (требуется новый модуль)**

Модуль «Карта Коргау» не входил в scope текущей задачи. Однако архитектура TB-Expert **напрямую поддерживает** его добавление:

- Создание новой доменной сущности `KorgauCard` аналогично `Incident` (структура из ТЗ: id, date, obs_type, org_id, category, description, resolved).
- Репозиторий, команды импорта и запросы аналитики подключаются по шаблону существующих модулей.
- Механизм тегирования (`cause_categories`, `violation_types`) уже обкатан на актах расследований — тот же подход применим к наблюдениям.

**Оценка: 3-4 дня** на полный модуль (сущность, модель, репозиторий, импорт CSV, аналитика, API-эндпоинты).

---

#### F-11: Система алертов при превышении порогов
**Статус: Инфраструктура готова, логика не реализована**

Событийная архитектура TB-Expert (`DomainEvent` → `Mediator.publish_event()` → `EventHandler`) **спроектирована именно для этого**:

- Доменные события уже определены: `IncidentCreatedEvent`, `EnquiryActUploadedEvent`, `AnalysisRequestedEvent`.
- Для алертов достаточно: создать `KorgauAlertEvent`, реализовать `AlertEventHandler` с пороговой логикой.
- 4 уровня алертов из ТЗ (критический, высокий, средний, низкий) реализуются как enum + пороговые SQL-запросы.

**Оценка: 2-3 дня** (пороговая логика + хранение алертов + API-эндпоинт).

---

#### F-12: Рейтинг организаций по уровню соответствия
**Статус: Частично реализовано**

Агрегация `by_company` в аналитике уже позволяет ранжировать организации по числу инцидентов, типам травм, числу пострадавших. Для полноценного рейтинга необходимо добавить нормализованный скоринг с учётом данных Карт Коргау.

---

#### F-13: Корреляция нарушений в Карте Коргау с реальными инцидентами
**Статус: Не реализовано (требуется модуль Коргау)**

Однако сервис `ActMatchingService` уже реализует **аналогичную задачу** — корреляцию актов расследований с инцидентами по 5 измерениям (дата, имя, компания, регион, тип травмы) с взвешенным скорингом. Тот же подход (fuzzy matching + SQL-агрегация) применим для корреляции наблюдений Коргау с инцидентами.

---

#### F-14: Генерация предупреждений при росте нарушений
**Статус: Не реализовано (зависит от F-11)**

Реализуется как расширение системы алертов (F-11) с добавлением тренд-анализа (сравнение текущего периода с аналогичным прошлого года).

---

#### F-15: AI-классификация наблюдений по кластерам
**Статус: Паттерн реализован (применение к Коргау — нет)**

LLM-тегирование актов расследований (`cause_categories`, `violation_types`, `industry_tags`) — это **ровно тот же паттерн**, который нужен для классификации наблюдений Коргау по кластерам (СИЗ, работа на высоте, LOTO и т.д.). Промпты и схема извлечения адаптируются за 1 день.

## 5. Заключение

Система TB-Expert покрывает **порядка 60-65% функциональных требований** хакатона в готовом виде, включая все высокоприоритетные требования Компонента A (кроме предиктива). Ключевые преимущества текущей реализации:

1. **Боевая архитектура** — Clean Architecture с CQRS, async I/O, DI-контейнер, event-driven. Новые модули подключаются без рефакторинга.
2. **LLM-инфраструктура** — OCR, извлечение структурированных данных, генерация отчётов, RAG-поиск по НПА — всё работает.
3. **Полная модель данных** — 58 полей инцидента, 48 полей акта расследования, рекомендации со статусами, NPA-база с векторными эмбеддингами.
4. **Работающий дашборд** — Chart.js визуализации, фильтрация по 27 параметрам, DOCX-экспорт.

Основные доработки для полного соответствия ТЗ хакатона:
