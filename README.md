# Эксперт по ТБ

Бэкенд-система для управления охраной труда на предприятиях Республики Казахстан.

Основные возможности:

- учёт происшествий и инцидентов на производстве;
- импорт журналов из Excel-файлов (SAP-формат, одиночные файлы и ZIP-архивы);
- загрузка и обработка актов расследования (PDF, DOCX) с извлечением текста через OCR/LLM;
- аналитический дашборд с тепловой картой Казахстана;
- генерация AI-отчётов на основе собранных данных.

Стек: Python 3.12, FastAPI, SQLAlchemy (asyncpg), PostgreSQL + pgvector, Alembic, Punq DI, Pydantic v2, Chart.js.

## Установка и локальный запуск

### Предварительные требования

- [Docker](https://docs.docker.com/get-docker/) и [Docker Compose](https://docs.docker.com/compose/install/)

### Шаги

1. **Создайте файл окружения** — скопируйте шаблон и при необходимости отредактируйте значения:

   ```bash
   cp dev/env.example dev/.env
   ```

   Ключевые переменные:

   | Переменная | Назначение | Значение по умолчанию |
   | --- | --- | --- |
   | `POSTGRES_DB` | Имя базы данных | `tb_expert` |
   | `POSTGRES_USER` / `POSTGRES_PASSWORD` | Учётные данные БД | `tb_expert` / `tb_expert` |
   | `OPENAI_API_KEY` | Ключ для LLM-сервиса | — (обязательно) |
   | `OPENAI_BASE_URL` | URL LLM-сервера | `https://litellm-cloud.sk-ai.kz` |
   | `AUTH_ENABLED` | Включить аутентификацию | `true` (для разработки установите `false`) |
   | `NETWORK_ENV` | Окружение (`test` включает debug) | `test` |

2. **Запустите контейнеры:**

   ```bash
   cd dev/
   docker-compose up -d
   ```

   Будут подняты три сервиса:
   - **agent-postgres** — PostgreSQL 15 с расширением pgvector (порт `65432`);
   - **minio** — S3-совместимое хранилище файлов (API: `9000`, консоль: `9001`);
   - **agent-api** — приложение FastAPI (порт `8001`).

   Миграции базы данных применяются автоматически при старте контейнера (через `entrypoint.sh`).

3. **Проверьте работоспособность:**

   ```bash
   curl http://localhost:8001/health
   ```

   Ожидаемый ответ:

   ```json
   {"status": "healthy"}
   ```

   Дополнительно доступен эндпоинт `/readiness`, возвращающий `{"status": "ready"}`.

### Запуск без Docker

```bash
pip install -r requirements.txt
alembic upgrade head
uvicorn src.presentation.api.main:create_app --factory --host 0.0.0.0 --port 8000 --reload
```

Убедитесь, что PostgreSQL и MinIO запущены и переменные окружения настроены.

## Доступ к Swagger

После запуска интерактивная документация API доступна по адресу:

```text
http://localhost:8001/docs
```

Swagger UI позволяет просматривать все эндпоинты, их параметры и схемы запросов/ответов, а также выполнять запросы прямо из браузера.

Если `AUTH_ENABLED=false` (рекомендуется для локальной разработки), авторизация не требуется — все запросы выполняются от имени тестового пользователя `dev-user`.

Основные группы эндпоинтов:

| Группа | Путь | Описание |
| --- | --- | --- |
| Incidents | `/v1/incidents/` | CRUD инцидентов, импорт журналов |
| Acts | `/v1/acts/` | Акты расследования, загрузка и поиск |
| Analytics | `/v1/analytics/` | Дашборд, тепловая карта, AI-отчёты |
| Files | `/v1/files/` | Загрузка файлов в S3-хранилище |

## Демо-страница

Демо доступно по адресу:

```text
http://localhost:8001/demo
```

Это одностраничное приложение на HTML/JS (Chart.js), которое взаимодействует с API и демонстрирует все ключевые функции системы. Страница содержит четыре вкладки.

### Вкладка «Главная»

Приветственный экран с заголовком «Мониторинг происшествий» и тремя карточками навигации:

- **Данные** — переход к журналам инцидентов и актам расследования;
- **Аналитика** — переход к интерактивному дашборду;
- **Отчёт** — переход к генерации аналитических отчётов.

### Вкладка «Данные»

Содержит две подвкладки: **Журналы** и **Акты**.

**Журналы** — таблица инцидентов с возможностью фильтрации и импорта:

- фильтры: диапазон дат, статус расследования (завершено / не завершено), результат расследования, компании, регионы;
- кнопки «Применить» и «Сбросить» для управления фильтрами;
- кнопка «Загрузить журналы» — импорт Excel-файлов (.xlsx / .xls) с данными об инцидентах;
- таблица с колонками: дата, компания, регион, вид происшествия, ДЗО, тип травмы, статус;
- пагинация (кнопки «Назад» / «Вперёд»).

**Акты** — таблица актов расследования:

- фильтры: диапазон дат, тип акта, компании, регионы;
- кнопка «Загрузить акты» — загрузка PDF или DOCX файлов;
- таблица с колонками: дата акта, компания, регион, пострадавший, тип акта, язык;
- пагинация.

### Вкладка «Аналитика»

Интерактивный дашборд с визуализацией данных:

- **KPI-карточки** в верхней части:
  - «Всего происшествий» — общее количество инцидентов;
  - «Пострадавших» — количество пострадавших;
  - «Погибших» — количество летальных исходов.

- **Круговые диаграммы:**
  - «По виду происшествия» — распределение инцидентов по классификации;
  - «По типу травмы» — распределение по тяжести травм.

- **Тепловая карта Казахстана** — интерактивная SVG-карта всех регионов:
  - цветовая шкала от зелёного (мало инцидентов) до красного (много);
  - отмечены крупные города: Астана, Алматы, Шымкент;
  - при наведении — всплывающая подсказка с количеством инцидентов в регионе.

Фильтры аналогичны вкладке «Данные»: даты, статус, результат, компании, регионы.

### Вкладка «Отчёт»

Генерация аналитических отчётов:

- **«Сводная статистика»** — формирует текстовый отчёт на основе агрегированных данных (без обращения к LLM);
- **«AI-отчёт»** — генерирует расширенный аналитический отчёт с использованием LLM (анализ причин, оценка рисков, рекомендации);
- фильтры: диапазон дат, компании, регионы;
- после генерации доступны:
  - кнопка «Скачать .docx» — экспорт отчёта в формат Word;
  - кнопка «Скопировать Markdown» — копирование текста отчёта в буфер обмена.

## Troubleshooting

### Порт 8001 уже занят

Измените проброс портов в `dev/docker-compose.yaml`:

```yaml
ports: [ "8002:8000" ]  # замените 8001 на свободный порт
```

### Контейнер `agent-api` не запускается

Проверьте логи:

```bash
cd dev/
docker-compose logs agent-api
```

Частые причины:

- отсутствует или некорректен файл `dev/.env` — убедитесь, что он создан из `dev/env.example`;
- PostgreSQL не успел запуститься — Docker Compose ждёт healthcheck, но при первом запуске инициализация БД может занять время. Перезапустите: `docker-compose restart agent-api`.

### Миграции базы данных не проходят

```bash
cd dev/
docker-compose logs agent-api | grep ALEMBIC
```

Если ошибка связана с уже существующими таблицами — возможно, база осталась от предыдущего запуска. Для полного сброса:

```bash
docker-compose down -v   # удалит volumes с данными
docker-compose up -d
```

### MinIO не запускается

Убедитесь, что порты `9000` и `9001` свободны. Консоль MinIO доступна по адресу `http://localhost:9001` (логин: `minioadmin`, пароль: `minioadmin`).

### AI-отчёт не генерируется

- Убедитесь, что в `dev/.env` заданы корректные значения `OPENAI_API_KEY` и `OPENAI_BASE_URL`;
- Проверьте доступность LLM-сервера из контейнера:

  ```bash
  docker-compose exec agent-api curl -s https://litellm-cloud.sk-ai.kz/health
  ```

- Увеличьте таймаут при необходимости: `LLM_REPORT_TIMEOUT=120.0`.

### Swagger не открывается

- Проверьте, что контейнер запущен: `docker-compose ps`;
- Попробуйте `http://localhost:8001/docs` (без `/api/...` префикса — `ROOT_PATH` применяется внутри, но локально порт проброшен напрямую).
